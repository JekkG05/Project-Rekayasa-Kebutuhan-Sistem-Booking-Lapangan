# Pengujian Aplikasi

## Tabel Pengujian Aplikasi

| **Aspek Kualitas**     | **Tujuan Pengujian**                          | **Metode Pengujian**                      | **Hasil Pengujian**  | **Status**     |
|------------------------|-----------------------------------------------|-------------------------------------------|----------------------|----------------|
| **Fungsionalitas**      | Memastikan fitur utama seperti booking lapangan berfungsi dengan baik. | Menguji fitur pemesanan lapangan di aplikasi. | Booking dapat dilakukan dengan sukses. | Lulus |
| **Kinerja**             | Memastikan aplikasi berjalan dengan lancar tanpa lag. | Menguji kecepatan aplikasi dalam proses pemesanan dan load halaman. | Aplikasi merespon dalam 2 detik. | Lulus |
| **Keamanan**            | Memastikan data pengguna terlindungi.         | Melakukan pengujian terhadap proteksi data menggunakan HTTPS dan enkripsi data. | Data pengguna aman. | Lulus |
| **Pengalaman Pengguna (UX)** | Memastikan aplikasi mudah digunakan dan nyaman dilihat. | Uji coba dengan pengguna untuk melihat kemudahan navigasi dan antarmuka. | Pengguna puas dengan UI/UX. | Lulus |
| **Kompatibilitas**      | Memastikan aplikasi berjalan dengan baik di berbagai perangkat. | Uji aplikasi di perangkat Android dan iOS serta browser web yang berbeda. | Aplikasi berjalan baik di semua perangkat. | Lulus |
